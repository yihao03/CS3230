Question 1

This is a variation of the knapsack problem. Observe that each element could be in either set XXX or set YYY, giving rise to the idea that we can use a recursion where in each function, we take the minimum between putting current element A[i]A[i]A[i] in set XXX or set YYY.

taking the top down approach,

class Solution:
  int[n][nM] memo
  int sum

  function minDiff(A)
    initialise memo to -1
    sum <- sum of elements in A
    return helper(len(A) - 1, sumX)

  function helper(i, sumX)
    if (memo[i][sumX] != -1)
      return memo[i][sumX]
    
    if (i < 0)
      memo[i][sumX] <- abs(sumX - 2 * (sum - sumX))
      return memo[i][sumX]

    memo[i][sumX] <- min(helper(i - 1, sumX + A[i]), helper(i - 1, sumX))
    return memo[i][sumX]

Question 2

Basic recursion

fun maxNonAdj(A, n)
  if (n < 0)
    return 0
  if (n == 0)
    return A[0]

  // third case is for when the current element is negative
  return max(maxNonAdj(A, n - 1), A[n] + maxNonAdj(A, n - 2), maxNonAdj(A, n - 2))

Bottom up approach

fun maxNonAdj(A, n)
  int[n] dp
  dp[0] <- A[0]
  for i in range(1, n)
    twoago <- i - 2 >= 0 ? dp[i - 2] : 0
    dp[i] <- max(dp[i - 1], A[i] + twoago, twoago)
  
  return dp[n - 1]

Correctness

At each step, there's two cases:

  • current element is positive: we can either take it or not take it, and we take the maximum of the two
  • current element is negative: we do not considering taking it, and we take the maximum of the sum including previous elemtent and the sum excluding previous element

Time and space complexity

The time complexity of this algorithm is O(n)O(n)O(n) since we only need to iterate through the array once. The space complexity is also O(n)O(n)O(n) since we need to store the maximum sum of non-adjacent elements up to each index in the dp array. However, we can optimize the space complexity to O(1)O(1)O(1) by only keeping track of the maximum sums for the previous two indices instead of the entire dp array.

fun maxNonAdj(A, n)
  int[2] dp
  dp[0] <- A[0]
  for i in range(1, n)
    twoago <- i - 2 >= 0 ? dp[0] : 0
    val <- max(dp[1], A[i] + twoago, twoago)
    dp[0] <- dp[1]
    dp[1] <- val
  
  return val

Question 3

This is just knapsack with an additional parameter, r where r is the number of objects that can be placed in the knapsack. We use a 3D array to memoize the results.

class Solution:
  int[n][m][r] memo
  int[n] W
  int[n] V
  
  function knapsack(W, V, m, r)
    initialise memo to -1
    this.W <- W
    return helper(len(W) - 1, m, r)

  function helper(i, m, r)
    if (i < 0 || m <= 0 || r <= 0)
      memo[i][m][r] <- 0
      return memo[i][m][r]

    if (memo[i][m][r] != -1)
      return memo[i][m][r]

    no_take <- helper(i - 1, m, r)
    if (W[i] > m)
      memo[i][m][r] <- no_take
    else
      take <- V[i] + helper(i - 1, m - W[i], r - 1)
      memo[i][m][r] <- max(no_take, take )

    return memo[i][m][r]
