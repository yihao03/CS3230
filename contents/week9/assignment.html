Question 1

This question is a simplified version of the meeting rooms problem. There is no constraint on overlapping security guards on the same days and the sole goal is to find the minimum number of security guards needed to cover all the days.

To do this, we can use a greedy approach. We sort the security guards by their starting day. We start by setting the curr_day to the starting day. Then, we iterate through the security guards to find the guard with the latest ending day bib_ibi​ where ai≤curr_daya_i \leq curr\_dayai​≤curr_day. We update the curr_day to bi+1b_i + 1bi​+1 and repeat the process until we have covered all the days.

Note that at any instance, if there is no guard ggg where ag≤curr_daya_g \leq curr\_dayag​≤curr_day, then it is impossible to cover all the days.

Question 2

This question seems almost like a knapsack problem, except we have to cover the entire duration [1,M][1, M][1,M], while minimizing the cost of the guards. We can use dynamic programming to solve this problem.

We can define an iterative algorithm that iterates from day 1 to M and build up the solution.

function minCost(guards, M)
  dp <- array of size M + 1, initialized to infinity
  dp[0] <- 0
  for i in range (1, M)
    dp[i] <- min(dp[a_j - 1] + c_j for all guards j where a_j <= i <= b_j)
  return dp[M]

Time complexity

We iterate from 1 to M, and for each day, we iterate through all the guards to find the guard with the minimum cost that covers the day, giving a time complexity of O(n⋅M)O(n \cdot M)O(n⋅M).
