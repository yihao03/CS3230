Comparison based algorithms

  • Comparison-based algorithms:
    • Elements can only be compared with each other: <, ≤, =, >, ≥
    • No other information of the elements can be used.

Stirling's approximation: For large n, n! can be approximated as:

log⁡(n!)≈nlog⁡n−nlog⁡e+O(log⁡n)⊆Θ(nlog⁡n) \log(n!) \approx n \log n - n \log e + O(\log n) \subseteq \Theta(n \log n) log(n!)≈nlogn−nloge+O(logn)⊆Θ(nlogn)

Proving the lower bound

In general, we can show that there is at least lll leaves in the decision tree, where l≥nl \ge nl≥n is the number of possible outcomes. Since in each comparison, we can only split the possibilities into ccc branches, the height of the tree must be at least log⁡cl\log_c llogc​l. Therefore, the number of comparisons must be at least log⁡cn\log_c nlogc​n.

In the case of sorting, we have n!n!n! possible outcomes (permutations), so the number of comparisons must be at least log⁡cn!≈nlog⁡n\log_c n! \approx n \log nlogc​n!≈nlogn.
